\subsection{Problema del Logaritmo discreto}
Il problema del logaritmo discreto è alla base di molti sistemi di cifratura e protocolli di crittografia.


\begin{definition}[Logaritmo Discreto]
    Per un gruppo moltiplicativo $(G,\cdot)$ con un elemento $\alpha \in G$ di ordine $n$ e sia $\beta \in \langle \alpha \rangle$, trovare un intero $a$ tale che $0 \leq a \leq n-1$ per cui 
    \[
        \alpha^{a} = \beta
    \]
    ovvero ($a = \log_{\alpha}(\beta)$).
\end{definition}

\vspace{1em}
La particolare utilità del problema è fondata dalla difficoltà di invertire l'operazione di elevamento a potenza nel gruppo.

\subsection{Crittosistema di ElGamal}
Il sistema crittografico di ElGamal è basato sul problema del logaritmo discreto nel gruppo $({\mathbb{Z}_p}^*,\cdot)$.

\begin{definition}[ElGamal]
    Sia $p$ un numero primo tale che il logaritmo discreto nel gruppo $({\mathbb{Z}_p}^*,\cdot)$ è intrattabile, sia $\alpha \in {\mathbb{Z}_p}^*$ un elemento primitivo.
    Il crittosistema ha $\mathcal{M} = {\mathbb{Z}_p}^*,\  \mathcal{C} = ({\mathbb{Z}_p}^* \times {\mathbb{Z}_p}^*)$, infine $\mathcal{K}$ è definito come segue
    \[
        \mathcal{K} = \{ (p,\alpha,a,\beta) \mid \beta \equiv \alpha^a \pmod p\}
    \]
    le variabili $p,\alpha,\beta$ formano la chiave pubblica e $a$ rappresenta la chiave privata.
    Per $K = (p,\alpha,a,\beta)$ e per un numero $k \in \mathbb{Z}_{p-1}$ casuale si definisce la cifratura
    \[
        e_K(x,k) = (y_1,y_2)
    \]  in cui 
    \begin{align*}
        &y_1 = {\alpha}^k \mod p\\
        &y_2 = x\beta^k \mod p.
    \end{align*}
    Per $y_1,y_2 \in {\mathbb{Z}_p}^*$ si definisce la decifratura
    \[
        d_K(y_1,y_2) = y_2({y_1}^a)^{-1} \mod p.
    \] 
\end{definition}

\noindent
In pratica, nel crittosistema di ElGamal, il \emph{plaintext} $x$ viene mascherato attraverso la moltiplicazione per $\beta^k$, $y_2 = x\beta^k$. Il valore $\alpha^k$ è trasmesso insieme al \emph{ciphertext} e chi è a conoscenza della chiave $a$ è capace di ricavare il valore $\beta^k$. Infine è possibile togliere la maschera al \emph{ciphertext} per ottenere $x$.

\vspace{1em}
Ovviamente il sistema è sicuro finché il valore di $a$ è sconosciuto, $a$ può essere calcolato con $a = \log_{\alpha}(\beta)$, proprio il problema del logaritmo discreto. Finché non esiste un algoritmo efficiente (polinomiale), capace di risolvere il problema del logaritmo discreto, il sistema è ritenuto sicuro, (per essere sicuri contro attacchi per enumerazione, è comunque consigliabile utilizzare $p$ primo di almeno 2048 bit).

\paragraph{Algoritmo Rho di Pollard}

Poiché $\langle \alpha \rangle$ è ciclico, il valore cercato può essere associato a un elemento in $\mathbb{Z}_p$, procedendo come per l'algoritmo Rho \ref{alg:rho} per la fattorizzazione, si cerca una sequenza $x_1,x_2,\dots$ applicando una funzione $f$, \emph{pseudo-casuale} nel momento in cui si ottengono due elementi tali $x_i =x_j \ i < j$, si ha la possibilità di calcolare il logaritmo discreto. Come nel caso della fattorizzazione si cercano due elementi $x_i = x_{2i}$ per risparmiare tempo.

\vspace{1em}\noindent
Sia $S_1 \cup S_2 \cup S_3$ una partizione di $G$ con $|S_1| \approx |S_2| \approx |S_3|$ si definisce una funzione $f: \left(\langle \alpha \rangle \times \mathbb{Z}_n \times \mathbb{Z}_n\right) \to \left(\langle \alpha \rangle \times \mathbb{Z}_n \times \mathbb{Z}_n\right)$ come segue:

\begin{equation*}
    f(x,a,b) = 
    \begin{cases}
        \ (\beta x, a,b+1) &\text{ se } x \in S_1\\    
        \ (x^2, 2a,2b) &\text{ se } x \in S_2\\
        \ (\alpha x, a+1,b) &\text{ se } x \in S_3 \\   
    \end{cases}
\end{equation*}

ogni tripla $(x,a,b)$ deve essere tale che $x = \alpha^a\beta^b$, ovviamente la tripla $(1,0,0)$ ha la proprietà, si osserva che se $(x,a,b)$ è valida allora anche $f(x,a,b)$ è valida, si definisce 

\begin{equation*}
    (x_i,a_i,b_i) = 
    \begin{cases}
        (1,0,0) &\text{ se } i = 0\\
        f(x_{i-1},a_{i-1},b_{i-1}) &\text{ se } i \geq 0
    \end{cases}
\end{equation*}
si confrontano le triple, finché $(x_{2i},a_{2i},b_{2i})$ e $(x_i,a_i,b_i)$ finché $x_i = x_{2i}$, quando accade
\[
    \alpha^{a_{2i}}\beta^{b_{2i}} = \alpha^{a_i}\beta^{b_i}.
\]
Sia $c = \log_\alpha(\beta)$  allora
\[
    \alpha^{a_{2i} + cb_{2i}} = \alpha^{a_i}+\beta^{cb_i}
\]
dal momento che $\langle \alpha \rangle$ ha ordine $n$ ne segue che 
\begin{align*}    
    a_{2i} + cb_{2i} \equiv a_i + cb_i \pmod n \\
    c(b_{2i} - b_i) \equiv a_i - a_{2i} \pmod n
\end{align*}

se $\gcd(b_{2i} - b_i,n) = 1$ (quindi esiste un'inversa moltiplicativa) allora
\[
    c = (a_i - a_{2i})(b_{2i}-b_i)^{-1} \pmod n .
\]

Si mostra un'esecuzione dell'algoritmo, si noti come si controlla che $1 \notin S_2$, poiché, in caso contrario, si otterrebbe continuamente la tripla $x_i = (1,0,0)$.

\paragraph{Metodo Index Calculus}
Gli algoritmi precedenti possono essere applicati a qualsiasi gruppo, il metodo dell'index calculus è applicato ai gruppi ${\mathbb{Z}_p}^*$ con $p$ primo e $\alpha$ è una primitiva modulo $p$, poiché specifico, l'algoritmo è generalmente più veloce. L'algoritmo è simile ad algoritmi di fattorizzazione, fa uso di una base di piccoli fattori primi $B = \{p_1,\dots,p_b\}$ chiamata \textbf{base di fattori} e opera come segue:

\begin{itemize}
    \item vengono calcolati i logaritmi discreti dei fattori nella base B
    \item viene calcolato il logaritmo discreto di $\beta$ facendo uso dei logaritmi nella base
\end{itemize}
Si descrivono più nel dettaglio le fasi. Sia $C$ un intero tale che $C > b$, nella fase 1 si costruiscono $C$ congruenze modulo $p$ della forma
\[
    \alpha^{x_j} \equiv {p_1}^{a_{1j}} {p_2}^{a_{2j}}\dots,{p_b}^{a_{bj}} \pmod p
\]
per ogni $j = 1,\dots, C$, e in modo equivalente
\[
    x_j \equiv a_{1j}\log_{\alpha}(p_1) + \dots + a_{bj}\log_{\alpha}(p_b) \pmod {p-1}.
\]
